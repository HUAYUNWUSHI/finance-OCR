基于 ONNX-OCR 引擎开发，全自动识别 PDF 单据顶部编号，支持自定义格式匹配、批量重命名，可视化参数调节，小白开箱即用，无需复杂配置。

✨ 核心功能

- 批量PDF处理：自动遍历指定目录，批量识别并重命名PDF文件，无需单个操作

- 双OCR引擎切换：支持印刷体（默认）、手写体两种识别模式，适配不同单据场景

- 自定义格式匹配：自由设定识别规则，支持字母-数字、数字-字母等组合格式，自动适配分隔符

- 可视化参数调节：对比度、检测阈值、识别阈值滑块调节，实时优化识别效果

- 预览功能：单文件预览识别区域增强效果，提前校验参数合理性

- 多线程高效运行：并发处理文件，提升批量处理速度，节省时间

- 完整日志记录：实时展示处理进度、成功/失败状态，失败原因清晰可查

- 安全重命名：自动处理重名文件，避免文件覆盖，原地重命名不移动文件

🧰 运行环境

- 操作系统：Windows 10/11（完美适配，其他系统未测试）

- Python版本：3.8~3.11（打包为EXE后，无需安装Python环境，直接运行）

- 核心依赖：OpenCV、PyMuPDF、ONNXRuntime、Pillow、Tkinter

🚀 快速上手（一键使用）

1. 准备工作

新建专属文件夹存放待识别PDF，推荐路径：

D:\Users\Administrator\Desktop\PDF待识别

将所有需要重命名的PDF文件放入该文件夹，确保文件清晰、正向无严重倾斜。

2. 启动工具

直接运行打包好的 pdf_ocr_renamer.exe 文件，工具自动初始化OCR引擎，无需手动安装依赖。

3. 基础配置

- 选择目录：点击浏览，选中上述存放PDF的文件夹

- 设置识别格式：默认格式为字母-数字，可自定义修改，例如：数字-字母、字母字母-数字数字等

- 选择OCR类型：印刷体单据选印刷体，手写编号选手写体

4. 开始处理

点击开始处理，工具自动逐文件识别，日志区实时同步进度，处理完成后自动停止，可随时点击停止中断任务。

🎛️ 高级参数调节

针对模糊、低清晰度PDF，可调节以下参数优化识别效果，调节后点击应用参数并重启OCR生效：

- 对比度：默认1.8，数值越高文字对比度越强，模糊单据可适当调高

- 检测阈值：默认0.15，控制文字区域检测灵敏度，数值越低检测区域越多

- 识别阈值：默认0.4，控制文字识别可信度，过滤低置信度识别结果

📷 预览功能使用

1. 点击预览区域的选择，挑选单个测试PDF

2. 点击预览，右侧展示识别区域增强后的效果图

3. 根据预览效果微调对比度等参数，确认无误后再批量处理

📂 文件命名规则

- 识别成功：直接以识别出的编号命名，例：A123.pdf、X789.pdf

- 重名文件：自动追加序号，例：A123_2.pdf、A123_3.pdf

- 仅处理后缀为.pdf的文件，其他格式文件自动跳过

⚠️ 重要注意事项

- 识别区域优先级：顶部25%区域 → 顶部50%区域 → 全页，优先识别单据顶部编号

- PDF文件需正向放置，倒置、严重倾斜会导致识别失败

- 批量处理过程中，请勿关闭工具或修改源文件，避免处理异常

- 工具默认使用CPU运行，无需GPU配置，低配电脑也可流畅使用

- 首次启动初始化OCR引擎稍慢，属于正常现象，后续启动速度会加快

- 临时图片文件会自动清理，不会残留垃圾文件

🐛 常见问题

- OCR引擎初始化失败：检查工具目录是否完整，勿移动exe文件和内置依赖文件夹

- 识别成功率低：调高对比度，调整识别格式，预览校验效果，确保PDF清晰

- 目录选择后无反应：确认目录路径无中文乱码、特殊字符，路径尽量简短
